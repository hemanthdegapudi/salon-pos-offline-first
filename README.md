# Salon POS System (Offline-First)

## Overview
A real-world Salon POS system designed for offline-first operation, enabling uninterrupted billing and transaction handling without internet dependency.

## Key Features
- Billing and receipt generation (PDF / print)
- QR-based UPI payment integration (pre-filled amount)
- Offline-first data handling
- Transaction failure recovery mechanism
- Role-based access control (RBAC)
- Search, pagination, and filtering

## Tech Stack
- Python (Flask / FastAPI)
- MySQL / SQLite
- SQLAlchemy

## Architecture
- Modular backend design (services, routes, models)
- REST API driven system
- Local-first persistence with recovery handling

## Current Status
Actively under development with focus on improving reliability and handling real-world edge cases.

## Planned Improvements
- Sync mechanism for multi-device usage
- Advanced error handling
- Reporting and analytics
